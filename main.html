Responsive Redesign

Context:

For my UI/UX class, we were tasked with practicing the workflow of redesigning a simple website.

I chose a restaurant in my hometown because I always thought its website was difficult to use and hard on the eyes.

I picked the home page of their website and analyzed it based on usability and accessibility standards. I then redesigned and built a new version of this webpage.

Evaluation:

I analyzed the usability of the website using criteria we learned about in class: usability, learnability, and memorability. Additionaly, I analyzed the accesibility of the website, using WebAIM WAVE and a screen reader. The results of this evaluation are shown below:

Usability:

  • Cluttered Information
  • Small Text Size and narrow letter spacing making it hard to read
  • Low Quality Photos
  • Poor Color Choices (color contrast, analogous colors)
  • Lack of Visual Style
  • Inconsistent Layout
  • Excessive Tabs

Accesibility:

  • 11 Errors from WebAIM Wave
  • Missing alt text for images
  • Low Contrast
  • Empty Links
  • Voiceover unable to read because of empty headers
  • Redundant Title Text
  • Redundant Links

Redesign:

I created low-fidelity wireframes, high-fidelity prototypes, and a visual design style guide before redesigning the website. I used Balsamiq to create a total of 3 wireframes of my screen, one for each of the screen sizes and used Figma to create 3 high-fidelity prototypes of my screen. I then annotated the lofi and hifi prototypes with more detail for creating the actual website.

The main differences in the prototypes are: more contrast, text hierarchy, greater text size to help with readability, alt descriptions and no "empty" headers or links for screen readers, better organization, and a preview of the lunch and dinner menu with a link to the pdf for full menu so that readers are able to quickly gauge if they want to eat at the restaurant (instead of having to open the pdf to see the disorganized menu). Overall, the prototypes also are more visually appealing due to simpler color schemes and better typeface.

Observations:

  • The site looks best on computers, but responds well to tablets and phones.
  • The text is much more readable
  • The screen reader works more effectively, especially with the alt text
  • The WebAim Wave reader got a much better score: 1 error
  • A user can make a quicker decision by glancing at the highlighted menu options instead of having to download the pdf
  • The text becomes larger and easier to read for tablets and phones, but the images need additional scaling.

Conclusion:

There is still much to be improved for this site, like better responsiveness and redesigning the entire website instead of just a snippet (although I did make quick changes to the about us and visit pages (click to view). Still, I think that these minor changes are big improvements and create a much more professional tone for the restaurant. More importantly, the website is much more accesible and usable.
